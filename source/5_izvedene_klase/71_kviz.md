# Квиз: апрстрактне и запечаћене класе

## Питање 1

```{mchoice}
:answer1: Апстрактне класе су класе које не могу бити инстанциране.
:answer2: Запечаћене класе су класе које не могу бити инстанциране.
:answer3: Из апстрактне класе не могу се изводити друге класе.
:answer4: Из запечаћене класе не могу се изводити друге класе.
:correct: 1,4

Који су од наведених исказа тачни?
```

## Питање 2

```{mchoice}
:answer1: У класи која је изведена из апстрактне, нису имплементиране све методе апстрактне класе.
:answer2: У апстрактној класи дефинисана је неапстрактна метода.
:answer3: Апстрактна класа је изведена је из апстрактне класе.
:answer4: Креирана је инстанца апстрактне класе.
:correct: 1,4

У којим случајевима ће компајлер јавити грешку током компајлирања?
```

## Питање 3

```{mchoice}
:answer1: Апстрактна класа не сме да садржи поља.
:answer2: Апстрактна класа не мора да садржи апстрактне чланове.
:answer3: Сви чланови апстрактних класа морају бити апстрактни.
:answer4: Апстрактна класа може да садржи и апстрактне и неапстрактне чланове.
:correct: 2,4

Који су од наведених исказа тачни ако се говори о апстрактним класама?
```

## Питање 4

```{mchoice}
:answer1: Не може се користити као базна класа.
:answer2: Мора да садржи запечаћене чланове.
:answer3: Из ње се не могу изводити друге класе.
:answer4: Не може бити изведена из друге класе.
:correct: 1,3

Који су од наведених исказа тачни ако се говори о запечаћеној класи?
```

## Питање 5

```{mchoice}
:answer1: Могуће је запечатити само методу која преоптерећује виртуелну методу.
:answer2: Када се једном метода запечати, више се не може преоптеретити.
:answer3: Запечаћивање метода негативно утиче на перформансе програма.
:answer4: Ако класа има запечаћену методу, онда се из ње не могу изводити друге класе.
:correct: 1,2

Који су од наведених исказа тачни, ако се говори о запечаћеним методама?
```

## Питање 6

```{mchoice}
:answer1: class A { }
:answer2: class A { private A(){ } }
:answer3: sealed class A { }
:answer4: class A { protected A(){ } }
:correct: 3

Једна од основних особина објектно оријентисаног језика је наслеђивање. Дате су
насловне линије дефиниције класе. Означи дефиницију класе која се НЕ може
наследити.
```

## Питање 7

```{mchoice}
:answer1: new
:answer2: virtual
:answer3: sealed
:answer4: override
:answer5: abstract
:answer6: base
:correct: 2,3,5

Да би наслеђени метод могао да се редефинише и тиме измени његова
функционалност у класама наследницама, у родитељској класи испред ознаке
повратног типа метода наводи се нека од понуђених кључних речи. Штиклирај
кључнe речи омогућавају редефинисање дефинисаног метода кроз ланац наслеђивања.
```
